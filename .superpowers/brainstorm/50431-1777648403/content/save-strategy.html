Draft-level save

One save button for the whole resume. Cleanest model, but user can lose unsaved edits from multiple sections at once.

Section-level save

Each section becomes a small editing workspace. Good balance between safety and simplicity, especially for repeatable blocks like experience and projects.

C. Item-level save
Tokopedia - Frontend Developer
Cancel
Save Item
Every company, project, certification, or award saves independently.

Item-level save

Most granular and safest per record, but the editor starts to feel form-heavy and repetitive very quickly.

Recommendation

  • Pick section-level save as the default interaction.
  • It avoids constant rerender pressure without turning every item into its own mini-form flow.
  • It also matches your intuition that “kalau standard semua di section”.

Main risk

  • If a section contains many items, saving the whole section may still feel broad.
  • That can be solved later with per-item edit dialogs inside a section, without changing the persistence model.
